## ドキュメント使用フロー

### 開発開始時
1. `requirements.md` で全体像を把握
2. `tech-spec.md` で技術詳細を確認
3. `dev-guide.md` でコーディング規約を確認
4. `progress.md` で現在のタスクを確認

### 機能実装時
1. `requirements.md` で機能仕様を確認
2. `tech-spec.md` でAPI/DBスキーマを確認
3. `dev-guide.md` でコーディング規約に従って実装
4. `progress.md` で進捗を更新

### レビュー時
1. `dev-guide.md` で規約準拠を確認
2. `requirements.md` で仕様通りか確認
3. `progress.md` で進捗状況を更新

## ドキュメント更新ガイドライン

### requirements.md
- **更新タイミング**: 大きな仕様変更時のみ
- **更新者**: プロジェクトオーナー
- **レビュー**: 必須

### tech-spec.md
- **更新タイミング**: API追加、DB変更時
- **更新者**: 該当機能の実装者
- **レビュー**: 推奨

### dev-guide.md
- **更新タイミング**: 新しいパターン・規約追加時
- **更新者**: 誰でも
- **レビュー**: 推奨

### progress.md
- **更新タイミング**: タスク完了時、毎週
- **更新者**: 該当タスクの実装者
- **レビュー**: 不要
